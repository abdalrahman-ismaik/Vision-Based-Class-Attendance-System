## Technical Implementation

### Callback Pattern:
```dart
// In guided_pose_capture.dart
void _updateInstruction() {
  if (_currentPoseIndex < _requiredPoses.length) {
    final poseType = _requiredPoses[_currentPoseIndex];
    final instruction = _getPoseInstruction(poseType);
    
    setState(() {
      _currentInstruction = instruction;
    });
    
    // Notify parent about pose change
    widget.onPoseChanged?.call(_currentPoseIndex, poseType, instruction);
  }
}

// In registration_screen.dart
GuidedPoseCapture(
  onPoseChanged: (poseIndex, poseType, instruction) {
    setState(() {
      _currentPoseIndex = poseIndex;
      _currentPoseType = poseType;
      _currentPoseInstruction = instruction;
    });
  },
  // ...
)
```

### Dynamic Header Structure:
```dart
Column(
  children: [
    // Progress dots (5 dots)
    Row(
      children: List.generate(5, (index) => Dot()),
    ),
    // Pose name badge
    Container(
      child: Text(_getPoseDisplayName(_currentPoseType)),
    ),
    // Instruction
    Text(_currentPoseInstruction),
  ],
)
```

## Key Decisions

1. **Single Source of Truth:**
   - Pose information originates in `guided_pose_capture.dart`
   - Flows up to parent via callback
   - Header displays it dynamically
   - No duplication of state

2. **Separation of Concerns:**
   - Header: Shows what to do (pose name, instruction, progress)
   - Camera view: Shows the student (clean preview + capture button)
   - No mixing of instruction UI with camera overlay

3. **Progressive Disclosure:**
   - Only show capture button when ready
   - Show progress indicator during capture
   - Hide instruction card during capture

4. **Minimal Animation:**
   - Removed complex pulsing animations
   - Simple gradient button is sufficient
   - Cleaner, more professional look

## Architecture

### Component Hierarchy:
```
RegistrationScreen (Step 2)
  ├─ Dynamic Header (new)
  │   ├─ Progress Dots
  │   ├─ Pose Badge
  │   └─ Instruction Text
  └─ GuidedPoseCapture (simplified)
      ├─ Camera Preview
      ├─ Capture Button
      └─ Progress Indicator
```

### Data Flow:
```
GuidedPoseCapture
  ├─ _currentPoseIndex changes
  ├─ _updateInstruction() called
  ├─ onPoseChanged?.call(index, type, instruction)
  └─ RegistrationScreen updates header state
      └─ Header re-renders with new info
```
